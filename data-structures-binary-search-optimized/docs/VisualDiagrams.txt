# Binary Search Visual Diagrams (ASCII Art)

## 1. Standard Binary Search Example

Array: `[1, 3, 5, 7, 9, 11, 13]`
Target: `9`

**Step 1:** Initial state
Indices: `0  1  2  3  4   5   6`
Array:   `[1, 3, 5, 7, 9, 11, 13]`
         `L                   H`
         `mid = (0+6)/2 = 3`
`arr[3] = 7` (7 < 9) -> Search in right half

**Step 2:**
Indices: `0  1  2  3  4   5   6`
Array:   `[1, 3, 5, 7, 9, 11, 13]`
                        `L    H`  (L was mid+1 = 4, H stays 6)
         `mid = (4+6)/2 = 5`
`arr[5] = 11` (11 > 9) -> Search in left half

**Step 3:**
Indices: `0  1  2  3  4   5   6`
Array:   `[1, 3, 5, 7, 9, 11, 13]`
                     `L`
                     `H` (L stays 4, H was mid-1 = 4)
         `mid = (4+4)/2 = 4`
`arr[4] = 9` (9 == 9) -> **Target Found at Index 4**

## 2. Find First Occurrence Example

Array: `[1, 2, 3, 3, 3, 4, 5]`
Target: `3`

**Step 1:** Initial state
Indices: `0  1  2  3  4  5  6`
Array:   `[1, 2, 3, 3, 3, 4, 5]`
         `L                   H`
         `ans = -1`
         `mid = (0+6)/2 = 3`
`arr[3] = 3` (3 == 3) -> Potential `ans = 3`, search left (`high = mid - 1`)

**Step 2:**
Indices: `0  1  2  3  4  5  6`
Array:   `[1, 2, 3, 3, 3, 4, 5]`
         `L       H`  (L stays 0, H was 3-1 = 2)
         `ans = 3`
         `mid = (0+2)/2 = 1`
`arr[1] = 2` (2 < 3) -> Search right (`low = mid + 1`)

**Step 3:**
Indices: `0  1  2  3  4  5  6`
Array:   `[1, 2, 3, 3, 3, 4, 5]`
               `L H`  (L was 1+1 = 2, H stays 2)
         `ans = 3`
         `mid = (2+2)/2 = 2`
`arr[2] = 3` (3 == 3) -> Potential `ans = 2`, search left (`high = mid - 1`)

**Step 4:**
Indices: `0  1  2  3  4  5  6`
Array:   `[1, 2, 3, 3, 3, 4, 5]`
         `L`
               `H`  (L stays 2, H was 2-1 = 1)
         `ans = 2`
`low (2)` is now greater than `high (1)`. Loop terminates.
**Result: First Occurrence at Index 2**

## 3. Search in Rotated Sorted Array Example

Array: `[4, 5, 6, 7, 0, 1, 2]`
Target: `0`

**Step 1:** Initial state
Indices: `0  1  2  3  4  5  6`
Array:   `[4, 5, 6, 7, 0, 1, 2]`
         `L                   H`
         `mid = (0+6)/2 = 3`
`arr[3] = 7`
`arr[L]=4`, `arr[mid]=7` -> Left half `[4,5,6,7]` is sorted.
Target `0` is NOT in `[4,7]` (`0 < 4`). So target must be in right unsorted half.
`low = mid + 1 = 4`

**Step 2:**
Indices: `0  1  2  3  4  5  6`
Array:   `[4, 5, 6, 7, 0, 1, 2]`
                           `L H` (L was 4, H stays 6)
         `mid = (4+6)/2 = 5`
`arr[5] = 1`
`arr[L]=0`, `arr[mid]=1` -> Left half `[0,1]` is sorted.
Target `0` IS in `[0,1]` (`0 >= arr[L]` and `0 < arr[mid]`). So search in this left sorted half.
`high = mid - 1 = 4`

**Step 3:**
Indices: `0  1  2  3  4  5  6`
Array:   `[4, 5, 6, 7, 0, 1, 2]`
                           `L`
                           `H` (L stays 4, H was 4)
         `mid = (4+4)/2 = 4`
`arr[4] = 0` (0 == 0) -> **Target Found at Index 4**
