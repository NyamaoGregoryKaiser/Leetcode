# Visual Diagrams for Bit Manipulation

This document provides ASCII art diagrams to help visualize how bitwise operations work.
Assume 4-bit numbers for simplicity in diagrams unless specified otherwise.

## 1. Bitwise AND (&)

Sets a bit to 1 if *both* corresponding bits are 1. Otherwise, it's 0.
Useful for checking if a bit is set, or for masking.

Example: `5 & 3`

  ```
  Decimal: 5
  Binary:  0101
  
  Decimal: 3
  Binary:  0011
  
  -----------------
  Operation: AND (&)
  Result:    0001   (Decimal: 1)
  
  Explanation:
  Bit 3: 0 & 0 = 0
  Bit 2: 1 & 0 = 0
  Bit 1: 0 & 1 = 0
  Bit 0: 1 & 1 = 1
  ```

## 2. Bitwise OR (|)

Sets a bit to 1 if *at least one* of the corresponding bits is 1. Otherwise, it's 0.
Useful for setting a bit.

Example: `5 | 3`

  ```
  Decimal: 5
  Binary:  0101
  
  Decimal: 3
  Binary:  0011
  
  -----------------
  Operation: OR (|)
  Result:    0111   (Decimal: 7)
  
  Explanation:
  Bit 3: 0 | 0 = 0
  Bit 2: 1 | 0 = 1
  Bit 1: 0 | 1 = 1
  Bit 0: 1 | 1 = 1
  ```

## 3. Bitwise XOR (^)

Sets a bit to 1 if *only one* of the corresponding bits is 1. If both are 0 or both are 1, it's 0.
Useful for toggling a bit, finding unique elements (as `A ^ A = 0`), or checking if bits are different.

Example: `5 ^ 3`

  ```
  Decimal: 5
  Binary:  0101
  
  Decimal: 3
  Binary:  0011
  
  -----------------
  Operation: XOR (^)
  Result:    0110   (Decimal: 6)
  
  Explanation:
  Bit 3: 0 ^ 0 = 0
  Bit 2: 1 ^ 0 = 1
  Bit 1: 0 ^ 1 = 1
  Bit 0: 1 ^ 1 = 0
  ```

## 4. Bitwise NOT (~)

Inverts all the bits of a single operand. 0 becomes 1, 1 becomes 0.
For `int` (32-bit signed), `~n` is equivalent to `(-n - 1)` due to two's complement.

Example: `~5` (assuming 4-bit for simplicity, but remember `int` is 32-bit)

  ```
  Decimal: 5
  Binary:  0101
  
  -----------------
  Operation: NOT (~)
  Result:    1010   (Decimal: -6, in 2's complement for 4-bit)
  
  Explanation (for 32-bit int):
  5  = 000...0101
  ~5 = 111...1010 (which is -6 in two's complement)
  ```

## 5. Left Shift (<<)

Shifts bits to the left by a specified number of positions. New positions on the right are filled with 0s.
Equivalent to multiplying by powers of 2 (if no overflow).

Example: `5 << 1`

  ```
  Decimal: 5
  Binary:  0101
  
  -----------------
  Operation: << 1
  Result:    1010   (Decimal: 10)
  
  Explanation:
  0101 becomes 1010. The leftmost '0' is shifted out. A '0' is added to the rightmost position.
  ```

## 6. Signed Right Shift (>>)

Shifts bits to the right by a specified number of positions. New positions on the left are filled with the *sign bit* (MSB).
If the number is positive, 0s are filled. If negative, 1s are filled.
Equivalent to dividing by powers of 2 (truncating towards negative infinity).

Example: `5 >> 1`

  ```
  Decimal: 5
  Binary:  0101
  
  -----------------
  Operation: >> 1
  Result:    0010   (Decimal: 2)
  
  Explanation:
  0101 becomes 0010. The rightmost '1' is shifted out. A '0' (sign bit) is added to the leftmost position.
  ```

Example: `-5 >> 1` (assuming 4-bit, -5 is `1011` in two's complement)

  ```
  Decimal: -5
  Binary:  1011
  
  -----------------
  Operation: >> 1
  Result:    1101   (Decimal: -3)
  
  Explanation:
  1011 becomes 1101. The rightmost '1' is shifted out. A '1' (sign bit) is added to the leftmost position.
  ```

## 7. Unsigned Right Shift (>>>)

Shifts bits to the right by a specified number of positions. New positions on the left are *always* filled with 0s.
Treats the number as an unsigned value. Useful for bit manipulation where the sign is irrelevant or for truly unsigned contexts.

Example: `5 >>> 1` (same as `5 >> 1` because positive)

  ```
  Decimal: 5
  Binary:  0101
  
  -----------------
  Operation: >>> 1
  Result:    0010   (Decimal: 2)
  ```

Example: `-5 >>> 1` (assuming 32-bit, -5 is `1111...1011`)

  ```
  Decimal: -5
  Binary:  11111111111111111111111111111011
  
  -----------------------------------------
  Operation: >>> 1
  Result:    01111111111111111111111111111101 (Decimal: 2147483645)
  
  Explanation:
  All bits shift right. The leftmost position is filled with '0', effectively making the number positive.
  ```

## Common Bit Manipulation Patterns Visualized

### Brian Kernighan's Algorithm: `n & (n - 1)`

Removes the least significant set bit (rightmost '1').

Example: `n = 12` (binary `1100`)

  ```
  n       : 1100
  n - 1   : 1011
  -----------------
  n & (n - 1): 1000   (Decimal: 8)
  
  Explanation:
  The rightmost '1' in 'n' at position 2 (0-indexed) is cleared.
  All bits to its right (which were '0' in 'n') become '1' in 'n-1'.
  All bits to its left remain unchanged (or follow standard subtraction rules).
  When ANDed, the rightmost '1' position becomes '0', and all bits to its right are '0' because (0 & 1 = 0).
  ```

### Check if Power of Two: `(n > 0) && ((n & (n - 1)) == 0)`

A power of two has exactly one '1' bit (e.g., 1, 2, 4, 8, ...).
`n & (n - 1)` will be 0 if and only if `n` had only one set bit.

Example: `n = 8` (power of two)

  ```
  n       : 1000
  n - 1   : 0111
  -----------------
  n & (n - 1): 0000 (Result is 0) -> TRUE
  ```

Example: `n = 6` (not a power of two)

  ```
  n       : 0110
  n - 1   : 0101
  -----------------
  n & (n - 1): 0100 (Result is not 0) -> FALSE
  ```
