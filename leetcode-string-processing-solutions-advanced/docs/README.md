## Problems Covered

This project includes solutions and analysis for the following classic string manipulation problems:

1.  **Longest Palindromic Substring**
    *   **Description:** Given a string `s`, return the longest palindromic substring in `s`.
    *   **Approaches:**
        *   Expand Around Center (Optimal)
        *   Dynamic Programming
2.  **Minimum Window Substring**
    *   **Description:** Given two strings `s` and `t`, return the minimum window substring of `s` such that every character in `t` (including duplicates) is included in the window.
    *   **Approach:** Sliding Window with Hash Maps (Optimal)
3.  **String to Integer (atoi)**
    *   **Description:** Implement `myAtoi(string s)`, which converts a string to a 32-bit signed integer according to specific rules (whitespace, sign, digits, overflow).
    *   **Approach:** Careful Parsing with Overflow Checks (Optimal)
4.  **Group Anagrams**
    *   **Description:** Given an array of strings `strs`, group the anagrams together.
    *   **Approaches:**
        *   Using Sorted String as Key (Hash Map / Map)
        *   Using Frequency Array as Key (Hash Map with Custom Hasher)

## Getting Started

### Prerequisites

*   A C++17 compatible compiler (e.g., GCC 7+ or Clang 5+)
*   CMake (version 3.10 or higher)
*   Git (optional, for cloning the repository)
*   **Google Test:** For running unit tests.
    *   Most package managers have it (e.g., `sudo apt-get install libgtest-dev` on Ubuntu, then `sudo apt-get install cmake build-essential` and `sudo cmake .. && sudo make` in `/usr/src/googletest/googletest`).
    *   Alternatively, CMake can fetch it if `find_package(GTest CONFIG REQUIRED)` is used and GTest is installed in a discoverable location or `GTEST_DIR` is set.
    *   **Simplified:** CMake's `FetchContent` can download and build GTest and Benchmark automatically, but it's typically better to have them pre-installed for cleaner builds if possible. For simplicity, this project expects them to be found by `find_package`.
*   **Google Benchmark:** For running performance benchmarks.
    *   Similar installation process to Google Test.

### Building the Project

1.  **Clone the repository (if applicable):**
    ```bash
    git clone https://github.com/your-username/string_manipulation_project.git
    cd string_manipulation_project
    ```
    *(Note: This is a placeholder; you'll need to create your own repo)*

2.  **Create a build directory and navigate into it:**
    ```bash
    mkdir build
    cd build
    ```

3.  **Run CMake to configure the project:**
    ```bash
    cmake ..
    ```
    *(If GTest/Benchmark are not found, ensure they are installed or their paths are set. You might need to specify `CMAKE_PREFIX_PATH`)*

4.  **Build the project:**
    ```bash
    cmake --build .
    ```
    *(This will compile the main application, tests, and benchmarks)*

### Running Benchmarks

Performance benchmarks help compare the efficiency of different approaches.

```bash

```
*(The `--benchmark_min_time=0.1` flag sets the minimum time a benchmark runs to 0.1 seconds, which can speed up execution for very fast functions.)*
