## Project Overview

The project is structured to offer:
*   **Core Implementations (`src/`)**: Clean, optimal C++ solutions for 5 distinct binary search problems.
*   **Testing (`tests/`)**: A robust test suite using a simple custom framework to ensure correctness across various scenarios including edge cases.
*   **Documentation (`docs/`)**: Detailed explanations of the algorithm, problem descriptions, ASCII diagrams, and interview tips.
*   **Benchmarking (`benchmarking/`)**: Tools to measure and compare the performance of different approaches.
*   **Additional Implementations (`additional_implementations/`)**: Code demonstrating brute-force alternatives and discussing memory considerations.

## Build Instructions

This project uses CMake for building.

1.  **Clone the repository:**
    ```bash
    git clone https://github.com/your-username/binary_search_project.git
    cd binary_search_project
    ```
2.  **Create a build directory and configure CMake:**
    ```bash
    mkdir build
    cd build
    cmake ..
    ```
3.  **Build the project:**
    ```bash
    cmake --build .
    ```
    This will compile all executables: `binary_search_app`, `test_runner`, `benchmark_runner`, `brute_force_optimized`, `memory_versions`.

## Running Tests

After building, navigate to the `build` directory and run the `test_runner` executable:

```bash
cd build
./test_runner
```
The output will show the status of all test cases (PASS/FAIL) and a summary.

## Running Benchmarks

After building, navigate to the `build` directory and run the `benchmark_runner` executable:

```bash
cd build
./benchmark_runner
```
This will execute performance tests, comparing linear search against binary search and iterative versus recursive binary search for various input sizes.

## Main Algorithms

### Problem 1: Standard Binary Search
*   **Description**: Given a sorted array of integers and a target value, find if the target exists in the array and return its index. If not found, return an indicator (e.g., `std::nullopt`).
*   **Implementations**:
    *   `iterativeBinarySearch`: An iterative approach.
    *   `recursiveBinarySearch`: A recursive approach.
*   **Time Complexity**: O(log N) for both.
*   **Space Complexity**: O(1) for iterative, O(log N) for recursive (due to call stack).

### Problem 2: Find First and Last Occurrence
*   **Description**: Given a sorted array with potentially duplicate elements and a target value, find the index of its first and last occurrences.
*   **Implementations**:
    *   `findFirstOccurrence`: Modified binary search to find the leftmost index.
    *   `findLastOccurrence`: Modified binary search to find the rightmost index.
    *   `findFirstAndLastOccurrence`: Combines the above two.
*   **Time Complexity**: O(log N) for each.
*   **Space Complexity**: O(1) for each.

### Problem 3: Search in Rotated Sorted Array
*   **Description**: Given a sorted array that has been rotated at some pivot unknown to you beforehand (e.g., `[0,1,2,4,5,6,7]` might become `[4,5,6,7,0,1,2]`), find if a given target exists in the array.
*   **Implementation**: `searchInRotatedSortedArray`
*   **Time Complexity**: O(log N)
*   **Space Complexity**: O(1)

### Problem 4: Find Peak Element
*   **Description**: A peak element is an element that is strictly greater than its neighbors. Given an input array `nums` where `nums[i] != nums[i+1]`, find a peak element and return its index. Assume `nums[-1] = nums[n] = -infinity`.
*   **Implementation**: `findPeakElement`
*   **Time Complexity**: O(log N)
*   **Space Complexity**: O(1)

### Problem 5: Sqrt(x) - Integer Square Root
*   **Description**: Compute and return the square root of a non-negative integer `x`. Since the return type is an integer, the decimal digits are truncated.
*   **Implementation**: `mySqrt`
*   **Time Complexity**: O(log X) where X is the input number.
*   **Space Complexity**: O(1)
